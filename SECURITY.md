# Security Policy

## Supported Versions

Only the latest `main` branch is actively maintained.

## Reporting a Vulnerability

Do not open a public issue for security vulnerabilities.

Report security concerns privately to the repository owner. If GitHub private vulnerability reporting is enabled for this repository, use that feature.

Include:

- A clear description of the issue.
- Steps to reproduce.
- Impact and affected versions, if known.
- Any suggested fix, if available.

## Scope

This project is a local Android game. Security-sensitive areas include:

- Build and release workflows.
- Dependency updates.
- Local data persistence.
- APK signing or release configuration if added later.
